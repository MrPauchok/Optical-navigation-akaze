# О проекте

Данный проект предназначен для разработки системы визуальной навигации беспилотного летательного аппарата (БЛА) в условиях отсутствия или недоступности спутниковой навигации.

Основная идея проекта заключается в использовании камеры и заранее подготовленной картографической информации для определения текущего положения летательного аппарата. Перед началом полета выполняется обработка спутниковых изображений: карта разбивается на отдельные участки, для каждого участка формируется набор устойчивых визуальных признаков. Во время движения полученное с камеры изображение сравнивается с подготовленной базой данных, что позволяет определить координаты объекта относительно карты.

Проект использует методы компьютерного зрения, включая алгоритм выделения ключевых точек AKAZE, сопоставление изображений и фильтрацию ошибочных соответствий методом RANSAC.

## Назначение проекта

Система предназначена для:
- автономной навигации БЛА без использования GPS/GNSS;
- определения положения объекта по данным видеокамеры;
- предварительной подготовки картографических данных для снижения вычислительной нагрузки на бортовой компьютер;
- построения основы для дальнейшей разработки системы визуальной одометрии и навигации.

## Преимущества

- предварительная обработка карты позволяет уменьшить вычислительную нагрузку во время полета;
- возможность работы в условиях отсутствия спутникового сигнала;
- использование открытых картографических данных;
- модульная структура программного обеспечения;
- возможность адаптации под различные камеры и разрешения изображений;
- хранение признаков карты позволяет ускорить поиск соответствующего участка местности.

## Недостатки и ограничения

- точность локализации зависит от качества и информативности картографических данных;
- однородные поверхности (лес, поля, водоемы) содержат недостаточно устойчивых визуальных признаков;
- изменение освещения, времени года и погодных условий может ухудшать качество сопоставления изображений;
- подготовка большой территории требует значительных вычислительных ресурсов и времени;
- алгоритм требует наличия заранее подготовленной карты исследуемой области.

## Возможные направления развития

В дальнейшем возможно повышение надежности системы за счет:
- объединения визуальной локализации с данными инерциальных датчиков;
- использования нейросетевых методов сопоставления изображений;
- учета высоты и параметров движения БЛА;
- формирования нескольких вариантов карт для разных условий съемки.
## Структура проекта

### Директории

**features/**  
Каталог с предварительно рассчитанными визуальными признаками карты.  
Для каждого тайла хранятся ключевые точки, дескрипторы и дополнительные файлы для проверки качества обработки.

---

### Файлы проекта

**main.py**  
Основной файл запуска программы. Объединяет получение изображения, обработку данных и выполнение локализации.

**capture.py**  
Модуль работы с камерой. Выполняет получение кадров и настройку параметров видеопотока.

**capture.json**  
Конфигурационный файл параметров камеры. Содержит разрешение изображения, частоту кадров и другие настройки захвата.

**setup_capture.py**  
Вспомогательный скрипт для настройки и проверки параметров камеры.

**cut2.py**  
Скрипт подготовки картографических данных. Используется для разбиения исходной карты на отдельные изображения с сохранением привязки к координатам.

**tile_map.json**  
Файл структуры карты. Содержит информацию о доступных тайлах и используется для быстрого поиска необходимого участка изображения.

**tile_coords.json**  
Файл соответствия тайлов координатам местности. Используется для преобразования координат изображения в реальные координаты карты.

**extract_features_parallel.py**  
Основной скрипт формирования базы ключевых признаков карты.

Выполняет:
- предварительную обработку изображения;
- удаление областей растительности;
- бинаризацию;
- поиск ключевых точек AKAZE;
- расчет дескрипторов;
- сохранение результатов обработки.

**extract_features_parallel.py**  
Параллельная версия скрипта выделения признаков. Использует несколько потоков обработки для ускорения подготовки больших объемов картографических данных.

**localization.py**  
Основной модуль визуальной локализации.

Выполняет:
- получение изображения с камеры;
- поиск ключевых точек;
- сравнение с подготовленной базой признаков;
- поиск соответствующего участка карты;
- вычисление координат объекта.

**localization(comment).py**  
Версия алгоритма локализации с дополнительными комментариями для анализа и отладки кода.
